Hospitality Printing Troubleshooting

If you run into problems with printing KOTs, chances are that some setups are missing or incorrect. There are now three ways to print KOTs:

  • Print with the native Business Central method: Define the POS printers as OPOS or Windows (WinPrinter) and assign them to the Display Stations.
  • Send the KOTs to the Kitchen Service (KDS) that sends them to the Hardware Station to be printed. See How to: Set Up KDS Printers.
  • Use External Printing to bump KOTs from an Expeditor display station to a printer. See Expeditor Bump Printing.

These are the most common pitfalls:

Problem Action
POS External printer does not print a test page.
 If you click the Print Test Page action button when you are setting up a POS External printer and a blank dialog box appears, chances are that the POS printer is set as a Windows printer. Change the printer type to WinPrinter instead.
Nothing is printed to the External printer when the chit is bumped from the Display Station. Make sure that the Hardware Profile ID field in the POS External Printers configuration is blank. It must be empty when External Printer is used for printing after a bump.

If you are using an OPOS printer you cannot use the printer directly from Business Central. Since the External printing takes place on the service tier, the OPOS drivers will not load. The drivers are 32 bits but the service tier is 64 bit, and therefore the drivers will always fail to load.
Instead, configure the POS printer to use the Hardware Station for printing.

Make sure that the field Expeditor Bump Terminal No. is set to a POS in the same store as the Retail User is using.
Printing to the Hardware Station does not work. In the POS Printer setup, make sure that the port number of the Hardware Station is in the Print Server Host field as well. For example: localhost:8082.

Also make sure that the Printer Remote check box is not selected.

If nothing of the above helps solve your issues, examine the log files generated by the Hardware Station and the Kitchen Service.

The log file for the Hardware Station is located in the folder C:\ProgramData\LS Retail\LS Hardware Station\, filename LS_HWST_LOG.txt.

The log files for the Kitchen Service are located in the folder C:\ProgramData\LS Retail\Kitchen\Log.

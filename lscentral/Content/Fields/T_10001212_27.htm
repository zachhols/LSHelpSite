Order ID, Hospitality Type

This field indicates how the orders that belong to this hospitality type are identified. The options are:

  • Receipt No. Only - By default, the order is identified by its receipt number.
  • Dining Table ID - The order is marked with the ID of the dining table (as defined by the current dining area layout) that the order is for. Applies to a Dining Table Service service type only.
  • Manual Input by Sales Type - The order is marked by manual input that is requested when the order is created. The sales type has Request Description At Start of Transaction or On Posting.
  • Queue Counter - The order is marked with a queue counter. The queue counter is specified in the Queue Counter Code field.
  • Loading Card No. - The order is marked with a loading card number that is either scanned in or input manually when the order is created.
  • Delivery&Takeout Contact Name - The order is marked with the name of the contact the order is for. Applies to Delivery&Takeout in Restaurant and Call Center Service Type only.

The option selected in this field is fixed for the following Service Types:

  • Dining Table Service
  • Delivery&Takeout in Restaurant
  • Delivery&Takeout in Call Center
  • Loading Card Station

Note: Even if you have Order ID as queue counter, you can still let the sales type request a description to have the order marked with a queue counter and the name of the customer.

How to: Set Up Club Attributes

Every member club can have its own set of attributes. Attributes are extra information about the club's members. Some functions, Member Campaigns and Offers, for example, make use of the attributes for statistics and as triggers for offers and promotions. There are different types of attributes and they have different lookup types.

To set up attributes for club members

  1. Click the icon, enter Member Attribute, and select the relevant link.
  2. This page shows an overview of all attributes that you can specify for the club's members.
  3. Click the New action menu to specify a new set of attributes for the members in a club..
  4. Fill in the fields as described in the following table.
    FieldDescription
    CodeThis is the unique identifier for an attribute.
    DescriptionHere you can enter a description of the attribute.
    Visible TypeThe available options here are:
    Private. This attribute is displayed in the member's LS Loyalty app or in LS eCommerce when he or she is viewing his or her account information.
    System. This attribute is not displayed in the member's LS Loyalty app or in LS eCommerce when he or she is viewing his or her account information.
    Linked ToThe options here are Account, Contact, or Both. Depending on how you choose, the attribute will be available on the Member Account card, the Member Contact card, or on both cards.
    Attribute TypeThe available options are:
    Text. Use this type for text or code fields.
    Integer. Only integer numbers can be used.
    Decimal. The input must be numeric.
    Date. The input must be a valid date.
    Boolean. The input can be 0, 1, YES, NO, FALSE, TRUE.
    Lookup TypeThis field validates the input and provides lookup options. The lookup options are:
    None. No lookup or validation. The system will only validate the attribute type.
    Options. All valid inputs are inserted into the Member Attribute Lookup table (click Lookup Setup on the Related action menu.
    Range. This works the same way as Options, but the Attribute Lookup Table will only include 2 records, the lowest and the highest valid values. It is, for example, possible to have two entries, 1 and 99. This would allow all values between 1 and 99.
    Table. This will result in a lookup in the table you specify in the Table Setup ID column.
    Auto. Transfer. When you select this type, you must select the transfer from a table and which field you want to transfer. The transfer will be automatically maintained by the system.
    Dynamic Query. This lookup type is linked to a Dynamic Query. When the query runs, it will return a value, and that value will be the attribute value.
    Campaign. A system attribute. This lookup type is inserted automatically when you create a member campaign. The type has the same ID as the campaign.
    Table Setup ID You can only use this field if the lookup type is Table. Here you specify which dynamic lookup you wish to use to validate the input and lookup of values.
    Transfer from TableYou can only use this field if the lookup type is Auto. Transfer. Here you specify the table you want to transfer from. The options are:
    Member Account, Member Contact, and NAV Contact.
    Transfer Field IDThe number of the field you want to transfer automatically to the attribute table.
    MandatoryIf you select this check box, the system will automatically display the attributes in the Attribute Setup on the Member Account and Member Contact cards. The value specified in the Missing Attribute Handling field on the Member Management Setup card determines whether the system will display a warning or an error message if this attribute is not filled in.
    Allow BlankYou can only edit this field when the Lookup Type is None. This attribute tells the system whether the attribute value can be left blank.
    Default ValueIf you want the system to set an initial value when a record is inserted, you should specify this value here.
    Test InputThis field is only used to test the setup. Here you can test both the lookup and the validation of the input. This will work the same way as it does in the system.

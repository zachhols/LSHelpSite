Printing labels 

The system holds information about all labels needed in the future, thus for example making a print request both at the beginning date of a periodic offer and at the ending date.

Label orders are made to each store, item, variant and unit of measure. New labels to print should be made if:

  • Price changes
  • Comp. price change
  • Description changes (item or variant)
  • Barcode to use for item changes
  • Label function changes
  • New variant added
  • New unit of measure is marked to be printed

The label functions in the system are for reference. They should show prices up to 100.000 otherwise you will have to modify the functions.

Label Printer selection can be used to print on different printers.

Item setup

On the Item Card there is one setup field on the General tab:

  • Print Variants Shelf Labels. If selected, then Shelf Labels are ordered for each variant.

Unit of measure setup

You can let the system order labels for other units than the sales unit of measure. On the Item Units of Measure card, mark the Print Shelf Label field and add description for the label if needed.

Shelf label functions

To set up Shelf Labels and assign functions to them: 

  1. Click the icon, enter Shelf Label Functions, and select the relevant link.
  2. Enter a Label Code and assign a Codeunit to it.
    This code is the reference used in the system. You can change the Codeunit ID in the label functions and thus change the label printing for all labels with that specific label defined for them.

Item label functions

To set up Item Labels and assign functions to them: 

  1. Click the icon, enter Item Label Functions, and select the relevant link.
  2. Enter a Label Code and assign a Codeunit to it.
    This code is the reference used in the system. You can change the Codeunit ID in the label functions and thus change the label printing for all labels with that specific label defined for them.

Assigning shelf labels to item

To assign Shelf Labels to an Item: 

  1. Click the icon, enter Retail Item List, and select the relevant link.
  2. Click Navigate - Master Data - Labels Printing Setup - Shelf Label Setup .
  3. Assign labels to the item. 

You can define labels down to an individual store using the Store Group Code. The system chooses the most detailed setup. You can also define more than one label for each store and define quantity for each setup. 

If you define label in the Red Flag Label Code field the system will use that label if the price originates from Promotion rather than the price table.

The label setup on items has Text 1 and Text 2 for printing text on the labels. If the Boolean field Text 1 Is Item Descr. is selected, Text 1 is taken from the Item Card or the variant (if the label is for variant). If it is not selected for Text 1, any text written in the field overwrites the description from the item/variant. The same goes for Text 2; any text there is used for the label but if it is empty, Description 2 from the item or variant is used.

Assigning item labels to item

To assign Item Labels to an Item:

  1. Open the Retail Item Card.
  2. Click Navigate - Master Data - Labels and Printing - Item Label Setup.
  3. Assign Labels to the item. You can define labels down to an individual store using the Store Group Code; the system chooses the most detailed setup. You can also define more than one label for each store.

The label setup on items has Text 1 and Text 2 for printing text on the labels. If the Boolean field Text 1 Is Item Descr. is selected then Text 1 is taken from the Item Card or the variant (if the label is for variant). If it is not selected for Text 1 then any text written in the field overwrites the description from the item/variant. Same goes for Text 2, any text there is used for the label but if it is empty then Description 2 from the item or variant is used.

Shelf label printing

To print Shelf Labels:

  1. Click the icon, enter Shelf Label Printing, and select the relevant link.
  2. The Shelf Label Printing page shows the labels defined, quantity to be printed, and printed quantity for a single store on a single date. You can change the store and/or date, for example to print out Monday labels on Friday.

To print labels just click the Print button and the labels in the line you are in are printed. You can also reprint labels by going to the Actions tab and click Reprint Printed Labels.


On the Actions tab you can also generate label orders. There are four functions to generate labels:

  • Create Needed Labels Request. Runs through the Action Table and checks if any data has change that needs to be updated on label.
  • Create Labels by Item Request. Creates labels for items based on your selection.
    • Add filters and highlight items you want to create labels for.  Only highlighted items will be processed.
  • Create Labels by Purch. doc Request. Creates labels from the lines in chosen unposted Purchase Doc.
    • Add filters and highlight documents you want to create labels for.  Only highlighted documents will be processed.
  • Create Labels by Posted Purch. doc Request. Creates labels from the lines in chosen posted Purchase Invoice.
    • Add filters and highlight documents you want to create labels for.  Only highlighted documents will be processed.

Shelf label layout

One type of Shelf labels is included in the release for direct implementation or as a reference for designing your own layouts.

Item label printing

To print Item Labels:

  1. Click the icon, enter Item Label Printing, and select the relevant link.
  2. The Item Label Printing page shows the labels defined, quantity to be printed, and printed quantity for single store on single date.  You can change the store and/or date, for example to print out Monday labels on Friday.
  3. To print labels, click the Print action and the labels in the line you selected are printed. You can also reprint labels by clicking Actions - Functions - Reprint Printed Labels.

The Actions - Functions menu also lets you generate label orders. There are four functions to generate labels:

  • Create Needed Labels. Runs through the Action Table and checks if any data has change that needs to be updated on label.
  • Create Labels by Item.  Creates labels for items based on your selection.
    • Add filters and highlight items you want to create labels for.  Only highlighted items will be processed.
  • Create Labels by Purch. doc. Creates labels from the lines in chosen un-posted Purchase Doc.
    • Add filters and highlight documents you want to create labels for.  Only highlighted documents will be processed.
  • Create Labels by Posted Purch. doc. Creates labels from the lines in chosen posted Purchase Invoice.
    • Add filters and highlight documents you want to create labels for.  Only highlighted documents will be processed.

Item label layout

Quick printing - printing from the Item Card

Shelf and Item Labels can be printed straight from the Retail Item Card:

  1. Click Actions - Printing, and then click either Print Shelf Labels or Print Item Labels. In both cases the system opens a form showing what will be printed based on the setup for the item. You can modify the records, change the quantity, delete, or add as you wish.
  2. To print, just click Print and all labels on the form will be printed. 

Printing from Item Card sets up printing for all variants. To choose just one variant printing is also available from the variant framework printing only the record you are in. On the Item Variants Reg. page LS Retail > BackOffice > Product Design > Retail Items > Edit, On the Navigate tab click Variants > Variant Framework.  On the Actions tab click Print Shelf Labels or Print Item Labels.  Note: Item needs to be marked “Print Variants Shelf Labels” for this to work.

Printing from Retail Purchase Order

Item Labels can be printed straight from Retail Purchase Order page:

  • Click Actions - Print - Print Item Labels.

The system prints the Item Labels for the whole order and always for sales unit of measure. The setup from items is used so only items with Item Label setup are printed.  

Printing labels 

Tip: The Label Functions in the LS Central system are only an example of what a label code could look like and it needs to be redesigned for customers.

  • It is not possible to preview labels before printing with Zebra design labels.
  • Printed labels can be reprinted through the Actions - Reprint Printed Labels button.

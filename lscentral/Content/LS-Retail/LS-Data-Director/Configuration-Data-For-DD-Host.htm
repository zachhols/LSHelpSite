Configuration Data for DD Host

If you change any settings, click the Save Config button and then restart the Data Director Service to reload the configuration.

If the DD system is already running on a local host and you want to update the configuration directly, click the Get Config button and the configuration data will be loaded from the DD’s memory.

To send new configuration data back to a live DD system, click the Send Config button and the data will become active right away, and no restart is needed.

You can also connect to remote DD Host and fetch the configuration setup from the remote host, by selecting a host from the list, and clicking the Get Config button. You can change the settings, and then resend the configuration back to the host by clicking the Send Config button.
Tip: When the configuration is sent back, it will be reloaded live on the remote host, so there is no need for a restart on the remote host.

You can select more than one host in the list to get configuration data or an update: press the Shift or Ctrl key and hold while you select more hosts. All values that are the same for all selected hosts are open for editing.
Tip: You can also send DD and NAV license files simultaneously to multiple hosts.

You can also use this method, Get and Send Config, to update the Configuration data after upgrading DD to a newer version as there may be some new configuration values added that are not included in the older version.

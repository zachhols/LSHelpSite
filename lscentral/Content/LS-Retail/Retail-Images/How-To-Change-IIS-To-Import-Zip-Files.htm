How to: Change an IIS Setting to Import Large Zip Files

There is a functionality in Retail Images that lets you export and import retail images using a Zip file.

According to Business Central files of up to 350 MB in size can be imported without problems. You may, however, experience that you cannot import a retail image Zip file larger than 28 MB.

To solve that problem and import a Zip file larger than 28 MB, you must change a size setting in IIS (Internet Information Service):

  1. Open the IIS Manager as administrator:

  1. Select Microsoft Dynamics 365 Business Central Web Client:

  2. Select Request Filtering, and double-click.
  1. Click Hidden Segments, and then select and right-click web.config:

  2. Select Edit Feature Settings, and change the value in Maximum allowed content length (Bytes) to maximum, which is 4294967295 bytes, or to a lower value of your choice. The default value is 30000000 bytes.

  3. Click OK.

Activity Resource Groups

The resource groups are needed to define what types of resources are present in the system and which ones are required so that the system can calculate the availability of a product.

The resource groups determine the resource type and how capacity and usage is retrieved as the basis of availability.

To work with resource groups

  1. Click the icon, enter Activity Resource Groups, and select the relevant link.

    The columns on the page are:

    Column Description
    Code A unique ID reference for the resource group.
    Description General description of the resources belonging to this group.
    Type The available options here are: Human, Location, Equipment, Other.
    Capacity Based On Specifies how the system will figure out the capacity or number of resources:
    • Always Available: The resource is always available.
    • Staff Management Schedule: Based on the employee's roster schedule (from Staff Management). Only valid for human resources.
    • Opening Hours: Based on the opening hours of the activity location or store.
    • Location/Day/Time Specific Quantity: Based on quantity assigned on the resources per location, day, and hour. This setting will always try to search for the most specific setting (including exact time) in the resource day specific setup. If no setting is found on the exact time, then a zero (0 ) is returned.
      Note: If this setup is used, the time of the activity is always a requirement and this is in fact a specific slot availability.
    • Fixed Quantity: This setting will always try to search for the most specific setting in the resource day specific setup. If no setting is found, the default setting on the Resource card is returned. In this setup the time is not a required setting.
    • Resource Schedule: Based on time availability schedule assignment for the resource. Normally used for human resources when the user is not using Staff Management to retrieve employee schedules. The management of the resource schedule is accessible on the Activity Resources card.
    • Unit Tracking:Used in rental to calculate availability based on unit status.
    Usage Count Based On
    • No. of Activities Assigned: The number of activities assigned on the resource.
    • No. of People (in Activity): The number of people assigned on the activity.
    • Quantity (in Activity): The quantity entered on the activity.
    Usage Time Filtering
    • Resource Reserved Time: Retrieve the usage based on the time coverage of the activity.
    • Activity Start Time: Retrieve the usage based on the activitie's starting time.
    Capabilities The general capabilities assigned on the resource group.
    No. of Resources The number of resources assigned to this resource group.
    Time Restrictions In some cases there might be need to set Capacity Based On as setup not related to opening hours, but still opening hour restriction might apply. So this setting will also apply the opening hours restriction to any given capacity setting.
  1.  Click General Capabilities on the Home action menu to display the Resource Product Capabilities window where you specify the capabilities which all resources in the activity group will get by default.

    The capabilities are:

    ColumnDescription
    Activity TypeThe activity types the resources in this group can handle.
    ProductThe specific activity product the resources in this group can handle.

    If you leave this field blank, the resources can handle all products belonging to the specific activity type.

    Skill LevelThe default skill level assigned to the resources for the product.
    Selection PriorityThe default selection priority. Not relevant for the general capability settings.

3. Click the Unavailability action on the Home menu to assign and review unavailability for the whole resource group.

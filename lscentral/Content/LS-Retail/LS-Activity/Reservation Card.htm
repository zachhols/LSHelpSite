Reservations

The reservation is a container for multiple activity entries. The reservation types define how the Reservation Card looks and certain behavior of status and resource assignments.

To edit the Reservation Card from the Role Center

Reservation Transactions

See also

When shortcuts are used to create activity entries directly (for example a single appointment), the LS Activity solution always creates a reservation in the background and all activities belong to a reservation. This means that more activities can be added to that particular reservation, for example for a family or group activity, or for multiple activities within one reservation.

To edit the Reservation Card from the Role Center

  1. Click Reservations in the left navigation pane.
  2. Double-click an entry in the list to display the Reservation Card, or select an entry and click Edit on the Home action menu.

The table below lists and describes the fields on the Reservation Card's General FastTab:

Field Description
Reservation Type This field controls the definition of the reservation and what activities can be entered into a single reservation.
Client No. The main client or contact. Some reservation types may be multi-client (each activity has a client assignment), but on course reservations it may be the main internal contact as well, since the activity lines are keeping track of the participants (clients).
Name Only valid for certain types of reservations and used when creating Business Central invoices. Pulls in the assigned price and discount groups valid for retail items on the Additional Charges FastTab.
Description Description of the reservation. Automatically populated by the contact name, but can be edited by the user.
Period Normally, automatically updated by the activity entries, but for course reservations this field sets the starting date and time for the first session in the course.

Use the Assist Edit button to edit the date and time values directly, if needed. For other activity reservations, the period settings are automatically updated based on the dates and times of activities on the reservation. This update is done when the reservation or any activities are confirmed.
Location The activity location. Note: The value in this field filters all availability accordingly.
Language Code The client's language code. Note: Optionally, this setting decides the email template to be used for communication.
Price Group Retrieved from the customer or contact account. Sets the pricing group for additional charges.
Currency Group Retrieved from the customer or contact account. Sets the currency used for converting prices in Quotes and Invoices. The prices in the reservation are always shown the local currency. Note: If the Activity Setup - Price Currency is set, the prices in the reservation are in that currency.
Discount Group Retrieved from the customer or contact account.
Sales Person The sales person involved.
Main Internal Contact For internal reference.
Reference Additional field that can be used in the Role Center's search function.
E-mail The contact's email. Can be edited by the user. Note: If there is no client assigned to the reservation, then entering the email will search if the email exists in the member contact records and assign the related member contact as the client.
Mobile Phone No. The contact's mobile phone number. Can be edited by the user. Note: If there is no client assigned to the reservation, then entering the mobile phone will search if the phone number exists in the member contact records and assign the related member contact.
Comment Internal comment.
Additional Charge Limit The limit allowed to be charged additionally to the reservation at the POS.
Default Product No. For reservations that are mainly a single product entry, such as courses and classes, the user can specify the default product which will be used as a default product entry when creating new activity lines.

Note: For courses and classes, which have multiple sessions starting at the same date and time, it is necessary to specify a default product, which is then unique to each session. Otherwise, if assigning participants through the Role Center, POS, or web, the API will not be able to differentiate to which reservation the activity entry (participant) should be added.

Reservation Transactions

Any additional entries, for example deposits and deposit returns, can be viewed on the Reservation Transactions list with reference to the receipt number handling the transaction (on the Reservation Card, click Navigate - Entries - Transactions). All entries in this list will be included in the reservation balance summary. Also, both calculated and posted cancellation charges will be shown in this list. Any receipts that have been charged to the reservation from a POS will be shown in this view as well.

How to: Set up the Dining Reservation Desk

Menu and Interface Profile setup overview

To set up the Reservation Desk on a dedicated POS terminal

To set up the Reservation Desk via POS command

See also

The Dining Reservation Desk is a POS panel. You can set up the Dining Reservation Desk in two ways:

  • On a dedicated POS terminal - the POS terminal opens up showing the Reservation Desk. Reservations can be taken for multiple dining areas in multiple restaurants.
  • On the function menu (with a POS command) of the Dining Table Overview panel - with one button click, the reservation desk opens. Reservations can be taken for multiple dining areas within the restaurant.

Menu and Interface Profile setup overview

Reservation Desk on a Dedicated POS Terminal

 

 

Reservation Desk panel within the #ACT-DR profiles

 

 

 

#DR-FUNCTIONS menu contains Logon and Logoff

Menu #DR-DININGAREAS contains all dining areas that can be reserved

 The POS terminal has its own interface profile and menu profile.

The interface profile has the #ACT-DIN-RESERV-DESK startup controller.

The reservation desk panel is called #OFFLINE. All other controls and menus that belong to the panel have the #DR- prefix.

In demo data:

POS### is dedicated to the reservation desk. It belongs to restaurant S0005 but it takes reservations for the restaurant section of restaurants S0005 and S0010.

It uses the #ACT-DR menu and interface profiles. All menus and controls that belong to the reservation desk panel exist within the #ACT-DR profiles.

The #OFFLINE panel within interface profile #ACT-DR is the reservation desk panel.

The #DR-FUNCTIONS menu contains both LOGON and LOGOFF POS commands to log the staff on and off.

 

The #DR-DININGAREAS menu contains buttons for S0005-RESTAURANT and S0010-RESTAURANT dining areas.
Reservation Desk via POS Command

 

Reservation Desk panel within the ##DEFAULT profiles

 

#DR-FUNCTIONS menu contains Logoff

 

Menu #DR-DININGAREAS within the menu profile used by the restaurant. Contains all dining areas that can be reserved within the restaurant

 The Layout  function menu for the dining hospitality type contains a button with the POS command DR-SHOW_RESERV_PANEL with the reservation desk panel as a parameter - #DR-RESERV-DESK.

The reservation desk panel is called #DR-RESERV-DESK. All other controls and menus that belong to the panel have the #DR- prefix.

In demo data:

The ##DEFAULT interface profile and menu profile include the #DR-RESERV-DESK panel and all controls and menus that belong to it.

The #DR-FUNCTIONS menu contains only the LOGOFF POS commands (to close the reservation desk panel, does not log off the staff).

The Layout function menu in S0005 for the RESTAURANT has a button, Reservation Desk, with the DR-SHOW_RESERV_PANEL command and the parameter #DR-RESERV-DESK.

The #HOSP-FINE menu profile (used by S0005) contains the #DR-DININGAREAS menu which includes one button for the S0005-RESTAURANT dining area.

The Layout function menu in S0010 for the RESTAURANT has a button, Reservation Desk, with the DR-SHOW_RESERV_PANEL command and the parameter #DR-RESERV-DESK.

The #HOSP-COFF menu profile (used by S0010) contains the #DR-DININGAREAS menu which includes one button for the S0010-RESTAURANT dining area.

To set up the Reservation Desk on a dedicated POS terminal

  1. If necessary, export the #ACT-DR interface and menu profiles in demo data and import the #ACT-DR interface and menu profiles into the desired database.
  2. Create a POS terminal with #ACT-DR interface and menu profiles.
  3. Adjust the #DR-DININGAREAS menu so that it has a button for each dining area that is taking reservations on this terminal.

To set up the Reservation Desk via POS command

  1. If the #DR- menus and controls do not exist in ##DEFAULT profiles:
    1. Export the #ACT-DR interface and menu profiles in demo data and import the #ACT-DR interface and menu profiles into the desired database.
    2. Delete any controls or menus within the #ACT-DR profiles that do not have the prefix #DR- (if you are also setting up the desk on a dedicated POS terminal, create new menu and interface profiles and copy the #ACT-DR profiles to them, then delete any controls or menus that do not have #DR- prefix from the new profiles).
    3. Specifically, delete the #OFFLINE panel from the interface profile.
    4. Copy the menus from the #ACT-DR into ##DEFAULT.
    5. Copy the controls from the #ACT-DR into ##DEFAULT.
    6. Check the #DR-FUNCTIONS menu in the ##DEFAULT menu profile. Make sure it only has a LOGOFF button, not a LOGON button.
  2. Copy the #DR-DININGAREAS menu from ##DEFAULT into the menu profile that the restaurant you are setting up reservations for is running.
  3. Adjust the #DR-DININGAREAS menu within the restaurant menu profile so that it has a button for each dining area in the restaurant that is taking reservations.

Actions on the Activity Resource Card

This table lists and describes the actions on the Activity Resource Card's Settings and Relatedmenus.

Attributes Here the user can define attributes (additional field containers) that can be attached to the resources. These attributes are for information purposes only.
Capabilities This window displays the specific capabilities of the resources, skill levels, and selection priority for automatic selection of resources, or, in other words, which activity types and products the resource can perform.

The fields in the window are: 

  • Activity Type: The activity types the resource can perform.
  • Product: The specific product the user can perform. If you leave this field blank, the resource can perform all products in the specified activity type.
  • Skill Level: User-defined resource skill levels for the product.
  • Selection Priority: Priority (1-10) which is used for the automatic resource selection.
Day-Specific Capacity For certain resources, the availability (capacity) can be assigned based on location, weekday, specific date, and specific time. This could be products with limited availability for rides, trips, or shows. Also, this is used as a schedule for courses which start at a specific date.

This page can only be edited for resources with the capacity setting as Location, Day, or Time specific, either in the resource group or on the Resource card.
Employee Card Click this action to see details for the resource, if the resource is an employee.
Resource Status Click this action to view availability for this single resource.
Reservations Click this action to see a list of all resource reservations assigned to the resource.
Unavailability Opens the Resource Unavailability window where you can specify any time or date period where the resource may be unavailable.

The fields in the window are:

Season: Links to the season table to pick up related dates.

Weekday: Specify unavailability at a certain day of the week.

Date From and Date To: 

Time From and Time To: 

Comment: Here you can enter a comment which will be displayed on the resource availability views as an explanation of the unavailability.
Linked Resources Click this action to see if the resource is linked to another resource.

This is used in events, for example when you have a single resource divided into multiple resources (for example a meeting room that can be converted into multiple rooms). Here the user can assign the resources which should share the availability.
Rental Units Click this action to see if the resource has any rental units or to assign new rental units.

This is only used for resources which have capacity based on unit tracking. In that case the user must identify each single rental unit to be able to both assign and return specific units. Also, the capacity of the resource depends on the available units in this registration. Note: This action is only visible on resources with capacity based on unit tracking.
Availability adjustments Click this action to adjust availability on resources based on unit tracking.

This is only used for resources which have capacity based on unit tracking. The user can adjust future availability regardless if the units are registered or not. For example, if there is going to be a future purchase of equipment to be assigned as rental units, the user can increase the capacity temporarily without registering every single unit. However, at the specified date, the rental units must be registered to ensure they can be assigned to rental reservations.

Note: This action is only visible on resources with capacity based on unit tracking.

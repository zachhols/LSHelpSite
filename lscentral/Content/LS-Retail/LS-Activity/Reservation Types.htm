Reservation Types

The reservation types are containers for multiple activities or participant lists, for example for courses.

To set up reservation types

General FastTab

Deposits and Charging FastTab

Views FastTab

Group Settings FastTab

The Activity Types action

The user can define the view settings for the Reservations card and specify how dates are presented.

To set up reservation types

  1. Click the icon, enter Reservation Types, and select the relevant link.
  2. Click the New action.

Note: The Activity Types action opens a very important setting. Here the user is required to assign which activity types are allowed within the reservation and which view settings would be appropriate and suitable.

If this setting is ignored, you cannot assign any product on a reservation since the reservation type would give access to any type of product (based on the activity types allowed for the reservation).

The fields on the Reservation Type Card are:

General FastTab

Field Description
Code Unique ID reference for the reservation type.
Description The general description.
No. of Activity Types  Indicator showing the number of activity types allowed within the reservation type scope.
Multiple Clients Used when the reservation needs a client assigned to each activity in the reservation lines, or if the reservation only belongs to a single client assigned on the reservation header.
Multiple Locations Used when the reservation needs or allows location to be assigned on each activity in the reservation lines, or if location is only assigned on the reservation header.

Deposits and Charging FastTab

Field Description
Customer Account To show the Customer Account fields on the reservation header.

This option opens access to the Quote and Invoice functions on the Reservation Card's Action menu.
Invoicing Defines the invoicing options for this type of reservations: whether they can only be paid at the POS or be invoiced through standard Business Central, either one invoice for the whole reservation or an invoice for each activity line.
Deposit Account The item used for posting deposit payments on the POS.
Deposit Options Specifies whether deposits are accepted at Reservation level, Activity level (reservation line), or are not accepted at all.
Charge to Res (Tender Type) If you want to allow that the customer can charge any point of sale receipt towards the reservation (that is like room charging at a hotel), then this tender type will be used to balance the sale at the POS. Basically a reference to an interim G/L Account which keeps track of sales which haven’t been paid and are outstanding related to the reservation and will be finalized at later stage either through Business Centralinvoicing or at the point of sale. When the reservation is then either paid at POS or invoiced in Business Central, then any related POS receipt charges are added to the charged amount.
Default Charge Limit The default amount that is allowed to be charged additionally to a reservation from the point of sale. Note: This default amount is assigned on the reservation when it is created and can be edited on the reservation card.
Charge to Res Method Define how the charges to reservations are charged to the reservation. Either the charge can be sent as a single amount, or itemized per POS transaction line.
Charge to Res Item If the Charge to Res Method is Summary, the user must specify which item is going to be used as the posting carrier, and which is to be used as the finalizing item when the customer pays the reservation.

Views FastTab

Field Description
Date Range This defines how dates are represented on the Reservation Card and must take notice of the combination of activities within the reservation type. The options are:
  • Single Date: Reservations only show Date From. (Date To is not needed).
  • Date Period: Reservations show Date From and Date To for the period.
  • Date Schedule: No dates are shown on the reservation lines but are only assigned as schedule on the reservation header.
Show Activity Availability To display availability on each activity in the reservation lines.
Show Line Quantities To show quantity on the reservation lines.
Show No. of Persons To show the number of persons on the reservation lines.
Show Additional Charges To display the Additional Charges FastTab on the reservation lines.
Show Price & Amounts To show the price and amount related fields on the reservation lines.
Show Rental Units To have the Rental Unit assignment view open on the Reservation card and on the Activity card. Practical when the reservation type is presenting rental contracts. You can still assign and return rental units without having this view open by default, since the Unit Tracking view will be accessible on the action pane for rental activity products.
Show Time Range This field defines how times are represented on the Reservation Card. The combination of activities within the reservation type must be taken into notice. The options are:
  • Show From & To Times
  • Show From Times
  • Show No Times 
Show Comments Specifies if the Comment section should be open and accessible on the Activity Reservation Card.
Show Task Schedule Specifies if the Tasks section should be open and accessible on the Activity Reservation card.
Default Matrix View The default matrix template used for presenting graphical matrix.
Archiving Specifies if the reservations of this reservation type should be archived during the archiving process or if they should be excluded and always remain active.
Archive Older than X Days Specifies how many days in the past the activity entries should stay in the active transaction tables. If not specified, the default would automatically be set to 90 days. See Archive section for more details on the archiving process and logic.

Group Settings FastTab

Group Payment Handling Specifies if the payment process of the group should only pull in activity group lines of the charge type Group, or if they should include all, regardless of the charge type.
Default Charge Type This setting applies to group reservations only. Decides the default Charge Type assigned to the group reservation line when they are created initially. Charge type set as Group will set the related group charges to be paid by the group payment process, but if it is set as Individual, it will set the group line to be paid using normal payment processes for the individual Group member reservation.
Group Reservation Expanding This setting applies to group reservations only. Decides if the group lines are expanded to multiple activity reservations per group member, or if they are simply added to the paymaster reservation.

The Activity Types action

The Activity Types action opens the Reservation Type Activity card. Here the user is required to assign which activity types are allowed within the reservation and which view settings would be appropriate and suitable.

Note: The Activity Type is an important setting. If it is ignored, you cannot assign any product to the reservation, since the reservation type would give access to any type of product (based on the activity types allowed for the reservation).

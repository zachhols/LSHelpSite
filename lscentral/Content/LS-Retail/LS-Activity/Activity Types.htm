Activity Types

All products (services) that are sold through the LS Activity module must belong to an activity type.

To set up Activity Types

Pricing

Additional charges

See also

When an appointment or a reservation is created in the solution, all activity entries (for example an appointment, event, or course entry) must be assigned to a reservation.

Single appointments will always have a reservation as the container for the entry (that is be a line in a reservation), and more activity entries could theoretically be assigned to the same reservation (for example as a family or a group reservation).

Activity types describe how services of that type are entered (that is a single-date or multi-date activity) and various settings which should be generally applied to this type of entries.

To set up Activity Types

  1. Click the icon, enter Activity Types, and select the relevant link.
  2. Click the New action to display the Activity Type Card.

For information on how to set up default tasks for all products belonging to this activity type, see Activity Tasks.

General FastTab

Field Description
Code Unique ID reference for the activity type.
Description Description of the activities belonging to this type.
Interval Type The default interval settings for showing availability for these services.
Require Client No. If a Member Management contact reference is required for the activity assignment. If not, the user can still enter a name directly in the activity reservation for identification of the client, if needed.
Activity No. Series The number series used to assign a unique ID to the activity entries of that type. Can also derive from the default setting in the Activity Setup.
New Posting Item (Template) The retail item to use as a template when creating new activity products and a related retail item which defines the posting settings.
Product Template The activity product assigned which will be used as template or default settings for any new activity products within that activity type. When the user assigns an activity type on a new product, a prompt will state that a product template is assigned to the chosen activity type, and ask if the user wants to assign the default settings to the new product. Also, the task schedule and the resources required settings will be copied to the new product related settings. Note: Course and Class products will not inherit the resources required settings.
Day Duration Whether the activity is a single day or multiple day activity.
Tasks Enabled Specifies if the Tasks section is accessible on the Activity card.
No. of products Number of activity products belonging to the activity type.
Cancellation Policy The default cancellation policy setting that is assigned to the activities.

Availability FastTab

Field Description
Default Matrix View The default matrix template used to present the graphical matrix from the Activity card. Overwrites the reservation type default.
Advance Booking Limit How many days up front the user can make an activity reservation for this kind of service. Blank indicates no limitations.
Note: This setting can also be based on the client's Member Club assignment. See the Advance Booking Limits option on this card to define different limits based on the clien's tMember Club.
Show Non Availability (POS) Specifies if the POS availability lookup should show non-available times (which are fully booked) or if those times should be excluded from the listing. The purpose of displaying fully booked times is to let the user select fully booked times and assign them to a waiting list. If its not required to be able to assign to waiting list on the POS, then use the default setting of not showing non available times.
No. of Days in Availability How many days in advance the availability should be calculated and displayed when looking up availability for this kind of service in the Role Center and on the reservation lookups.

Note: This does not in any way limit the dates which a reservation can be assigned to, but it only controls how far up front calculations are needed in daily use.

You may need to limit this value to x number of days depending on the complexity of the products (that is how many resources must be checked to calculate availability).

This setting also defines how many days are calculated on the POS availability list.

Note: If the availability list is slow to appear on the POS, you may need to limit the number of days allowed in the view to speed up the process.

Allowances FastTab

Field Description
Pre-Sell Posting Item The posting item (retail item) used to post revenue when products in this activity type are presold (allowances).

Example: You can presell any item in bulk quantities, for example 10 spa sessions for a lowered price per session. A certain POS command, „presell”, is used to sell the products, and the product must have activity prices assigned with a quantity limitation over the quantity level of 1. If the Presell Posting Type field is set to Revenue at Consumption Date, then this setting is required so the system can pre-assign the revenue to a different product (and therefore to a different G/L account).
Pre-Sell Posting Type Defines how revenue is posted and handled when products (allowances) of this product type are presold.

Integration FastTab

Field Description
Table Mgmt. Integration Specifies whether the activity type entries should be synced to the table reservations in LS Hospitality.
Web Bookings Allowed Specifies whether the activity products belonging to this type should be available by default for booking through the web API. Activity product can overwrite this default.
Default Activity Card The page used to open the activity card. Leave blank if the system default should be used.

Emails FastTab

Field Description
Send Reminders (x Days Before) When to send an email reminder before an activity takes place. A blank field means that no reminders are sent.
Reminder Template The email template used for reminders to the clients.

Rental FastTab

Field Description
Late Return Handling None, Product Calculation, Fixed Amount. How the late return charges will be calculated. If by product calculation, then the late return activity line is re-recreated using the initial charge details, except the date and time settings of the line are adjusted to reflect the extended time and price calculated according to the same pricing settings. Fixed amount will add additional charge line with a fixed amount.
Late Return Charge Amount Only used if the late return handling is Fixed Amount.
Late Return (Flex Minutes) Flexibility (in minutes) allowed before late return charges are applied during returns.

Pricing

It is possible to assign general pricing and/or discount settings on all activity products belonging to a specific activity type. This pricing or discounting will only be activated, if the activity product itself does not have a price or discounts. The pricing or discounts on the product itself will always have priority, However, if the product comes up empty with pricing or discounting, the logic will check if the activity type of the product does contain any settings. This can be useful for assigning discounts as a general setting for a group of products, although the pricing might be based on the individual product.

See the Activity product pricingfor details on the pricing settings.

Additional charges

The activity type can be assigned additional charges that will be issued during the booking process. See the Additional charges setup on the activity product for further details on the settings available. Note: This feature is only enabled, if the Product Template has been assigned to the activity type, and actually, the default charges are the same as the ones set for that particular activity product. Note also that if the activity product has its own set of additional charges assigned, the default charges assignments from the product template setting is ignored.

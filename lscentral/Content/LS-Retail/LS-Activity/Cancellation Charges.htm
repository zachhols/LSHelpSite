Cancellation Charges

LS Activity supports the automatic calculation of cancellation charges, which can be adjusted depending on how many days or hours the client cancels in advance. Cancellation policies can differ between products.

Handling Cancellation Charges

When a reservation or activity reservation is canceled, the system checks if there is any cancellation policy assigned on the individual activity reservation. If there is, the number of days and hours until the reservation date is compared with the threshold settings of the cancellation policy. The system then determines if there is a cancellation fee to be charged. Note: No cancellation fee is applied, if the activity has already been paid.

In case a cancellation fee is calculated, an entry is inserted into the Reservation Transactions window (see the Reservation Card and the Activities Card), and will be added to the balance of the reservation. Note: This entry is only an estimated calculation and no finance posting has been issued at this point, therefore no transaction ID/reference is applied.

POS commands

There are multiple POS commands available to finalize the cancellation charge (or to cancel an activity from the POS). These commands serve two purposes: to finalize a canceled activity, or to cancel an activity and collect cancellation charges, if applicable.

These POS commands are:

  • CANCELACTIVITY - cancels a single activity and/or collects a fee.
  • CANCELRES - cancels all activities in a reservation and/or collects a fee.
  • CANCELCURRACT - same as above, but only for a currently active entry.
  • CANCELCURRRES - same as above, but only for a currently active reservation.

When the above POS commands are triggered at the POS from a POS button, any deposits related to the entries will be pulled into the POS for the purpose of returning to the customer and deduct from the cancellation charge.

Note: When the sale is posted and finalized on the POS, the transaction section of the reservation or activity will be showing the finalized cancellation charges, with navigation to the POS transaction (receipt).

How to: Set Up Regular Appointment

  • From the LS Activity Role Center you can access different setups by using the Administration menu in the left navigation pane.
  • From the Departments menu (LS Retail - LS Activity - Setup) you also have access to different setups.

The table below lists some of the setups that are available on the Setup page (under Administration).

Reservation Types Reservation type must be created as Single Date in Date Range. Other settings according to requirements.
Activity Types Activity type with Day Duration as a Single Day should be created, and within the scope of a reservation type (see the above setting, Reservation Type).
Resource Groups Your resources (persons) must be created, thus a resource group of Type Human. Capacity Based On would either be Staff Management Schedule, or Resource Schedule if staff management is not being used. Usage Count Based On Quantity (in Activity) would be a common setting. Usage Time Filtering should be Resource Reserved Time which would cover all activities assigned on the resource within a time frame.
Resources Each individual (or resource) must be created and assigned to the resource group. You must then maintain the persons’ availability using the Day Specific Capacity action, if the resource is using Resource Schedule as the basis of availability, where you assign the dates (or weekdays) the resource is available.
Product You must at least define one product which requires a human resource. Remember to assign the capabilities on the related resources. If the product is sold by the hour, then Quantity Conversion should be set to No. of Hours, thus any quantity entered would extend the duration of the appointment. Also, set Duration Type to Hour(s) and Default Quantity to the minimum reservation quantity (in this case number of hours).

If the user is allowed to change the time, then set Allow Time Change to Flexible, and Allow Quantity Change must be assigned (select the check box). On the POS, you may want to open a calendar and possibly the Quantity window, if the operator can change the number of hours. If the product requires more resources, you must define those and specify if they are manually selected or if the system automatically selects the next available during reservation.

Note: Make sure the resources needed are capable of handling the product.

There are two ways to make a resource capable of the product:

  1. On the Resource Groups page - You can define general capabilities for the whole group using the General Capabilities action (see the Resource Group List).
    You can assign the capability, either for a specific activity type (all activities of that type) or to specific activity product.

  2. On the Resource Card or List - Use the Capabilities action to assign the capabilities, either for all products in a specific activity type or down to a specific product.

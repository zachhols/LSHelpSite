LS Insight (2020.4) Release Notes

About This Release

Released - July 9, 2020

This version of LS Insight supports versions 8.4 and later of LS Central on-premises. Support for LS Central version 15.0 and later in cloud and hybrid cloud will be added in future versions of LS Insight.

This release was done to support the release of the LS Retail for Hotels extension. The LS Insight Hotels report has been updated and now includes a location dimension for property.

This version does not update older versions of LS Insight and should be set up from scratch.

Note: 
Since there are no changes to the standard LS Insight functionality, this version should only be used for new installations of LS Insight, or if you want to add LS Insight for Hotels to your current LS Insight setup.

The following is a list of changes and key features in LS Insight (2020.4).

New or Enhanced Features

Setup

Onboarding

We simplified the onboarding process and the onboarding documentation has been updated accordingly.

  • Source and Azure resource parameters are validated before resources are created in Azure.
  • Password prompts were removed.
  • Azure credentials are reused if a context exists in Power Shell.

Data warehouse

Database

The LS Insight database schema has been altered to include a new fact table for Hotels, fHotelStatistic.

A new stored procedure was added to populate the fHotelStatistic table, fHotelStatisticSP.

Pipelines

LS Central for Hotels

A new pipeline was added to the Azure Data Factory to run the Stored procedure, PL-SP-fHotelStatistic.

Reports

LS Insight Hotels

We made a few changes to the LS Insight Hotels report released with the last version of LS Insight.

  • New fHotelsStatistic table is now used in report
  • Location dimension that already existed in DW has been added to the report and connected to the Hotels model.
  • Two report pages showing the same data with different date parameters have been combined into one (Hotel Insights).

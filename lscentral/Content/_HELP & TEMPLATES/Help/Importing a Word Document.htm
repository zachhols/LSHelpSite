Importing a Word Document

Importing Word documents to Flare is not a complex task, but you should be aware of a few facts:

  • The cleaner your source files are, the smoother the import will be, meaning that the Word documents should ideally be formatted with styles and not local formatting.
  • Don't expect the import to be perfect; it won't be. No matter how clean your source files are, you will end up doing some formatting in your Flare project after the import.
  • Tables in the source files can prove difficult and will most likely require redoing in Flare.
  • You may want to clean up your source files before the import. This is often recommended, but it's a matter of opinion which option is more efficient, cleaning up the source files before the import, or checking and fixing the Flare topic files after the import; you would always want to check the Flare files after import, anyway.
  • If you are splitting the source document into multiple Flare topic files, you must decide where each topic should begin in the source file:
    • Generally, Heading 1 is used for a major topic (chapter level), and Heading 2 or Heading 3 for sub-topics.
    • Before you import files, choose a heading style at which to break topics.
      For example: if you choose Heading 2 style as the dividing point between topics, the content between each Heading 2 style in the source file will be converted into a separate Flare topic.
  • Local formatting in the Word document, for example bold texts or italics, is not imported into the Flare topic. You will have to go through the Flare topic and redo any local formatting.

 

About Word drawing objects and images

  • Flare puts imported images into sub-folders in your project's Resources\Images folder, in the Content Explorer.
  • Flare does not import drawing objects such as lines, and arrows.
  • Text in call-outs is imported as flat text.
  • Text in floating text boxes can be imported, but the text will be anchored instead of floating.

 

To import content from a Word document

  1. First of all, you must decide if you're going to
    a) create a new Flare project (step 2 below), or
    b) import your document into an existing project (step 3).
    (The LS Central Help, for example, is a separate Flare project named Online Help.flprj.)
  2. To create a new Flare project, click File - New Project - New Project to start the new project wizard. When you've created the project, continue with step 3.
  3. Make sure that the Flare project you want to import into is open with the Project Organizer in view.
  4. Right-click the Imports folder, and select Add MS Word Import File from the menu.
  5. Select New from template and MyWordImport under the Factory Templates folder.
  6. Type a file name. It is often practical to use the name of the document that you are importing.
  7. Click Add. The File Name you typed is listed in the Imports folder and the Word Import Editor opens.
  8. On the Source Files tab, and clear the Link Generated Files To Source Files check box. The check box is located in the bottom left corner of the window.
    Click Add Files (bottom right corner of the window), and select the Word document(s) you want to add (acceptable formats are DOC, DOCX, or RTF files).
    If you've selected multiple Word documents to import, you can change the order in which they are listed by clicking Move Up and Move Down.
  9. To split your document into multiple topics, do the following:
    1. Click the New Topic Styles tab. The Used Word Styles column on the left shows the styles that are used in the Word document you're importing.

    2. In the Used Word Styles column, double-click the style you want to start each new topic. The style appears in the right column.
      Your choice of style here would normally be Heading 1 or Heading 2, depending on the complexity and length of the imported document.

      Tip: At this point, don't worry too much about your choice. It is easy to redo the import with a different heading, if you're not happy with the result.

  10. Click the Stylesheet tab, and do the following:
    1. Make sure that the LSNav.css stylesheet appears in the Stylesheet field. If not, click the arrow and select LSNav.css from the drop-down list.
    2. Under Source Styles, select the option Don't Preserve MS Word styles.
  11. Click the Paragraph Styles tab, and map the Word styles to the corresponding Flare styles: select the Flare style in the right column (f. ex. h1) and click the Map button in the center of the window. Generally, it's sufficient to map the Word styles Heading 1 through Heading 3 to the Flare styles h1 through h3.

  12. To start the import, click Import (under the tab title, top left corner). Then click Yes when you're prompted to save changes to the Flare import file.

    The Accept Imported Documents dialog lists the documents to be imported (on the left) and a preview of the Flare topics (to the right).

  13. If you're satisfied with how the topic previews look, click Accept to import the files. The files are added to your Flare project.
  14. When the import finishes (it may take a while, depending on the size of the imported document), close the Word Import Editor.

Note: If you're not happy with how Flare has imported your document, for example how the document was split into topics, you can repeat the import as many times as you want using the same import file until you are satisfied with the result.

 

Where are the imported topics?

  • Flare put your imported topics together in a folder in the Content Explorer. That folder has the same name as the import file that you created.
  • Flare placed any imported images under the Resources folder, in a folder named the same as your import file.
  • Flare also created a TOC with the same name as the import file.

Note: After the import, you can rename folders and/or rearrange their location in the Content folder, as needed.
If you are adding topics to an existing project, instead of using the newly created TOC, you may want to expand and add the topics to the existing TOC.

Mix & Match - Price Group

The Price Group in the Mix & Match page determines the standard price for the item in the discount lines and also in which stores the discount is valid. The store must be a member of the Price Group for the discount to be valid.

Note: The standard price in the Discount line is not always the price that the discount is calculated on. The discount is always calculated on the best price that is active in the current Store.

Mix & Match - Store Groups

The Store Groups FastTab contains a reference to the Store Group that the item should be distributed to by replication.

In the demo database, the Table Distribution for Periodic Discount is All Default, which means that the table is distributed to and is valid in all retail locations, unless changed by the user.

When a Mix & Match discount is created, the system finds the Price Group that is linked to the distribution.

If you only want to replicate to a specific Store Group or a specific Store, you can change the store group.

Note 1: The Group only determines where the offer is replicated, not where it is valid, which the price group does. For example, if the price group is store 1 but the Store Group is ALL, the offer will be replicated to all stores but only be valid in store 1.

Note 2: Distribution is only valid, if the system is set up to replicate by Actions not Preactions.

The Mix & Match offers can be set up to only be valid for:

  • Customer Disc. Group - You can set up a discount only valid for customers who are members of specific Customer Disc. Group.
  • Loyality Scheme Code - You can set up a discount only valid for specific Loyalty cards belonging to a Loyalty Scheme.

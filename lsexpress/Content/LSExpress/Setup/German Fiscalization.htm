German Fiscalization

How to configure the Fiskaly connector for LS Express in Germany

Note: These configurations only apply to LS Express for Germany.

Connect your LS Express store to the Fiskaly server, the tamper-proof electronic recording and archive of business transactions, to fulfill the new regulations in Germany.

Create a Fiskaly Account

Create a new account with Fiskaly or Log in at https://dashboard.fiskaly.com/.

Note: Please contact Fiskaly sales for Fiskaly prices.

 

Create API Keys

To allow LS Express to connect to the Fiskaly cloud securely, you need to create credentials in Fiskaly.

 

Follow the steps below to create the credentials:

 

  1. Log in to your Fiskaly account at https://dashboard.fiskaly.com/.

  2. From the left side menu under “Developers”, select “API Keys”.

  3. Click the “CREATE API KEY” button.

  4. Type in a name for the “New API Key” and click the “CREATE” button.

  5. Copy your “API Key” and the “API Secret”, then paste it to Fiscalization Setup in LS Express (see next step).

     

Fiscalization Setup

In LS Express, find the “Fiscalization Setup” under Administration > Navigate > Fiscalization Setup.

 

  1. The Base Url should be "https://kassensichv.io/api/v1/".

  2. Paste the API Key and the API Secret that you copied in the last step into the fields under the “Connection” tab.

  3. Under the “General” tab, you can choose whether you want the data to be automatically uploaded to Fiskaly or if you want to upload it manually.

  4. Once you have configured the three fields above, test the connection by selecting Test Connection from the action's menu.

  5. If the connection is successful, you will see the message below:

Transactions

  1. The Transactions button will take you to the Transactions page, where all your store’s transactions will be listed.

  2. Once a transaction has been posted in the Point of Sale, the table will display the transaction entries. The Document No. is the Posted Sales Invoice number, the Transaction ID that corresponds to the Transaction ID in the Fiskaly server. Each transaction made in the POS has a unique transaction ID.

  3. View the transaction list in the Fiskaly dashboard.

  4. Each Point of Sale in your LS Express store has a unique TSS ID and a unique Client ID. You can see the TSS ID and Client ID that is automatically assigned to the POS under Administration > Navigate > Subscription.

     

    In this example, you can see that the user POS2 has the TSS ID 74bbc… which is the same as the TSS ID shown in the Fiskaly Dashboard screenshot above. And the Transaction ID also matches.

  5. The Cash Amount and Non Cash Amount columns show you how the payment is split between payment methods. In the example below, the highlighted line shows the customer paid with a Non Cash Payment method (Card), and therefore there is no entry in the Cash Amount column.

     

    In line 2, the customer paid with Cash only, and in line 3, the customer paid with a Card and got 2.71EUR back in Cash.

  6. Once a transaction has been finalized in the POS, LS Express will upload the payment information to the Fiskaly server. On a successful upload, the status field will change from “Pending” to “Completed”.

    If there is a problem with the upload, the status will say “Pending” or “Error”. If this happens, select the entry, and click the Upload button to resubmit the transaction.

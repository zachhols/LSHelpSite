Enable Posting to Customer's Account and Refund Payment

Note that if you leave the Customer POS Payment Nos. and Customer POS Refund Payment Nos. field blank and back to POS sales, the action "Pay to Customers Account" and "Refund Payment" will be disabled.

To enable the POS action "Pay to Customers Account" and "Refund Payment", you must select your Customer POS Payment Nos. and Customer POS Refund Payment Nos. which specifies the code for the number series used to assign document numbers to customer payments or refund payments posted at the POS.

 

  1. Select from the full list to see more details of the code.

  2. Select Lines.

  3. The setting should be done for both field Customer POS Payment Nos. and Customer POS Refund Payment Nos.. For example, CREFUND no. series line start with R00001. Then every time posted, the no.series line will increase automatically. You can specify any name and number for Starting No.

  4. Remember not to untick this box in Default Nos because this number series will be used to assign numbers automatically.
